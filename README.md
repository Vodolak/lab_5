
# Система контролю версій Git та співпраця через GitHub

=======
=======


## Мета заняття
- Освоїти основи роботи з Git як розподіленою системою контролю версій.
- Навчитися створювати локальні та віддалені репозиторії.
- Зрозуміти принципи роботи staging area та фіксації комітів.
- Навчитися створювати та використовувати гілки для розробки нового функціоналу.
- Опрацювати методи злиття гілок та розв’язання конфліктів.
- Здобути навички співпраці через GitHub (push, pull, pull request).
- Навчитися працювати з SSH-ключами та персональними токенами доступу.
- Використовувати `.gitignore` для захисту конфіденційних даних.


## Контрольні запитання та відповіді

### 1. Основи Git

**1. Яка різниця між Git та GitHub?**  
Git — система контролю версій.  
GitHub — онлайн-хостинг для Git-репозиторіїв.

**2. Що означає "розподілена" система контролю версій?**  
Кожен розробник має повну копію репозиторію.

**3. Різниця між git init та git clone?**  
git init — новий порожній репозиторій.  
git clone — копія віддаленого.

---

### 2. Staging Area та Commits

**1. Що таке Staging Area?**  
Проміжна зона для підготовки комітів.

**2. git add . vs git add -A**  
. — додає нові/змінені файли, не видалені  
-A — додає все, включно з видаленими

**3. Як написати гарне commit message?**  
Коротко, по суті, наказовий спосіб.

**4. Як виправити останній коміт?**  
git commit --amend

---

### 3. Гілки та злиття

**1. Що таке гілка?**  
Окрема лінія розробки.

**2. git merge vs git rebase**  
merge — злиття з новим комітом  
rebase — переписує історію

**3. Fast-forward merge**  
Злиття без нового коміту.

**4. Merge конфлікти**  
Виникають, якщо однакові рядки змінено в різних гілках.

**5. Чому не робити rebase публічних комітів**  
Бо переписується історія інших розробників.

---

### 4. Віддалені репозиторії

**1. Що таке origin?**  
Стандартна назва віддаленого репозиторію.

**2. git fetch vs git pull**  
fetch — тільки завантаження  
pull — fetch + merge

**3. git push --force — небезпечно**  
Може стерти чужі коміти.

**4. Як видалити віддалену гілку?**  
git push origin --delete branch_name

---

### 5. Скасування змін

**1. git reset --soft / --mixed / --hard**  
soft — залишає staging  
mixed — очищає staging  
hard — видаляє всі зміни

**2. git reset vs git revert**  
reset — переписує історію  
revert — новий коміт, що відкатує зміни

**3. Як скасувати git add?**  
git restore --staged file

**4. Як повернути видалений файл?**  
git checkout HEAD -- file

---

### 6. Безпека

**1. Що не має потрапляти в репозиторій**  
Паролі, токени, ключі, конфіденційні дані

**2. Якщо випадково закомітили пароль**  
Видалити, змінити пароль, очистити історію

**3. HTTPS vs SSH**  
SSH — безпечніше та зручніше

**4. Personal Access Token**  
Альтернатива паролю для GitHub

**1. Pull Request**  
Запит на злиття гілки з рев’ю

**2. Issues vs Pull Requests**  
Issues — задачі  
PR — зміни коду

**3. Fork**  
Копія чужого репозиторію для власної роботи

**4. GitHub Actions**  
Автоматизація CI/CD
